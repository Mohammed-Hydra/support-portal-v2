HYDRA-TECH Support Portal v2 – Deployment & Setup Steps

All steps for deploying and configuring the portal on Vercel. Use Print → Save as PDF to export this page as PDF.

1. Prerequisites

  • Git installed (git-scm.com/download/win)
  • GitHub account
  • Vercel account (vercel.com)
  • Project folder: D:\IT support portal v2

2. Initial Git Setup (if not a repo yet)

cd "D:\IT support portal v2"
git init
git add .
git commit -m "Initial commit - Support Portal v2"
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git
git branch -M main
git push -u origin main

Replace YOUR_USERNAME and YOUR_REPO_NAME with your GitHub details.

3. GitHub Repository

  • GitHub → New repository → Name (e.g. support-portal-v2), Public, do not add README
  • Create repository and use its URL in git remote add

4. Vercel Project Settings

Project → Settings → General → Build & Development

SettingValue
Root Directoryempty
Framework PresetOther
Build Commandnpm run build (Override: ON)
Output Directoryempty (Override: ON)
Install Commanddefault (Override: OFF)

5. Push and Deploy

cd "D:\IT support portal v2"
git add .
git status
git commit -m "Your message"
git push -u origin main

For later pushes use: git push

6. Fix 404 – Include Built Frontend

  1. Build: cd "D:\IT support portal v2\web" then npm run build
  2. Commit: git add web/dist web/.gitignore src/app.js then git commit -m "Include built frontend" then git push
  3. Vercel: Build Command = npm run build, Output Directory = empty, then Redeploy

7. Logo

Logo: web/src/assets/hydra-tech-logo.svg. Used in Layout and LoginPage via import.

8. Share Portal & Requester Links

Sidebar (admin/agent): Portal: Copy link, Requester: Copy link. Translations in web/src/i18n.js (EN/AR).

9. Company Name Column

Ticket Queue has a "Company Name" column after Contact. Data from API (requester_company_name / requester_company_from_contact).

10. GitHub Personal Access Token

When git push asks for password: GitHub → Settings → Developer settings → Personal access tokens → Generate (classic), scope repo. Use the token as the password.

11. Troubleshooting

IssueFix
404 NOT_FOUNDBuild Command = npm run build, Output Directory empty; commit and push web/dist; redeploy
Logo not showingEnsure web/src/assets/hydra-tech-logo.svg exists; redeploy
Cannot GET /loginBuild frontend (web/dist), set Build Command and empty Output Directory; redeploy
git not recognizedInstall Git, restart terminal
Repository not foundUse correct GitHub repo URL in git remote add

12. After Frontend Changes

cd "D:\IT support portal v2\web"
npm run build
cd ..
git add web/dist
git commit -m "Update frontend"
git push

Export as PDF: In this browser, press Ctrl+P (or Cmd+P), choose "Save as PDF" as the destination, then Save.
